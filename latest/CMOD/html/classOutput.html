Detailed Description

Process-global facade for accumulating and emitting CMOD output.

Output is purely static — there is one tree per CMOD run, rooted at top. Event-expansion code pushes nodes as it descends and pops as it ascends, building the same shape as the event tree itself. When the piece finishes, the tree is written to the project's .particel XML report so the user can audit every choice CMOD made.
