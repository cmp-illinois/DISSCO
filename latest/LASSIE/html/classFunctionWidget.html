Detailed Description

Base class for the per-function input widgets shown inside FunctionGenerator's stacked widget.

The legacy FunctionGenerator handles every CMODFunction inside one monolithic class: ~33 stacked-widget pages, ~100 slots, and matching branches in the XML read and write paths. This base class is the abstraction we're migrating to — each function becomes one subclass that owns its own UI, its own slots, and both directions of XML I/O.

Subclasses are registered with FunctionRegistry at startup and instantiated on demand by FunctionGenerator.
